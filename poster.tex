\begin{block}{1. Problem \& Motivation}
NVIDIA \cpasync{} (Ampere+) enables \emph{asynchronous} global$\to$shared memory copies. CUTLASS, FlashAttention-3, and Triton use multi-stage \cpasync{} pipelines; depth $S$ is a key tuning knob.

\vspace{4pt}
\textbf{The Problem:}
\begin{itemize}\setlength{\itemsep}{1pt}
  \item Practitioners tune $S$ on \emph{one} GPU and assume it transfers
  \item Effectiveness depends on \textbf{L2 capacity}, \textbf{BW/latency}, \textbf{SMEM} --- vary dramatically across GPUs
\end{itemize}

\vspace{3pt}
\coloredbox{ummaize!80!black}{ummaize!20}{%
\textbf{RQ:} Does optimal $S^*$ diverge across GPUs? Can a \emph{parameter-free} model predict this from architecture constants alone?%
}

\vspace{3pt}
\begin{center}

\begin{column}{0.50\textwidth}
\begin{block}{2. Parameter-Free Predictive Model}
Predict \cpasync{} speedup from architecture constants --- \highlight{zero fitted parameters}.

\vspace{3pt}
\textbf{Inputs per GPU} (pointer-chase microbenchmark): effective L2 capacity $C_{\text{L2}}^{\text{eff}}$, L2 latency $L_{\text{L2}}$, DRAM latency $L_{\text{DRAM}}$.

\vspace{3pt}
\coloredbox{umblue!70}{lightblue!50}{%
\vspace{-8pt}
\begin{align*}
h &= \min\!\bigl(1,\; C_{\text{L2}}^{\text{eff}} / W_{\text{conc}}\bigr) \quad\text{\small (L2 hit frac.)} &
L_{\text{eff}} &= h \cdot L_{\text{L2}} + (1{-}h) \cdot L_{\text{DRAM}} \\
T_{\text{comp}} &= \max\!\bigl(T_{\text{FLOP}},\; \mathbf{280}\bigr) \quad\text{\small (compute floor)} &
W_{\text{conc}} &= w_{\text{blk}} \!\times\! S \!\times\! \text{blks}_{\text{active}}
\end{align*}
\vspace{-10pt}
}

\vspace{2pt}
\coloredbox{ummaize!80!black}{ummaize!15}{%
\vspace{-4pt}
\[
\widehat{S}_{\text{pred}} = \underbrace{\frac{\max(T_{\text{comp}},\; L_{\text{eff}})}{\max\!\bigl(T_{\text{comp}},\; L_{\text{eff}}/S\bigr)}}_{\text{overlap benefit}} \times \underbrace{\left(\frac{\text{Occ}(S)}{\text{Occ}(1)}\right)^{0.15}}_{\text{occupancy penalty}}
\]
\vspace{-6pt}
}

\vspace{3pt}
\begin{itemize}\setlength{\itemsep}{1pt}
  \item \textbf{280-cycle floor}: without it stencil MAPE = 146\%; with it: \textbf{3.7\%}
  \item \textbf{$\alpha{=}0.15$}: occupancy loss highly sub-linear (3$\to$1 blk/SM $\approx$ 16\% loss)
\end{itemize}
\end{block}
\end{column}

\begin{block}{3. Hypotheses}

\coloredbox{supported!70}{supported!8}{%
\textbf{H1 -- L2-Capacity Driven} \hfill \textcolor{supported}{\textbf{SUPPORTED}}\\
Speedup inflects when $W_{\text{conc}} > C_{\text{L2}}$. \textit{\small L40S (112\,MB): minimal benefit; A40 (8\,MB): largest.}%
}

\vspace{3pt}

\coloredbox{partial!70}{partial!8}{%
\textbf{H2 -- Ridge-Point Driven} \hfill \textcolor{partial}{\textbf{PARTIAL}}\\
\textit{\small Ridge alone insufficient --- must combine with L2 + latency.}%
}

\vspace{3pt}

\coloredbox{supported!70}{supported!8}{%
\textbf{H3 -- Latency--Occupancy Tension} \hfill \textcolor{supported}{\textbf{SUPPORTED}}\\
$S^*$ balances latency hiding vs.\ occupancy loss. \textit{\small A40: non-monotonic $S^*(N)$.}%
}

\end{block}

\begin{column}{0.57\textwidth}
\begin{block}{8. Key Discoveries \& Conclusions}
\begin{enumerate}\setlength{\itemsep}{2pt}
  \item \textbf{Effective L2 $\neq$ Nominal L2.} H100: only \textbf{28\,MB} (not 50\,MB) due to L2 slice locality. First documentation of MIG L2 behavior.

  \item \textbf{280-cycle per-stage overhead floor.} Even 28 FLOP-cycle stencils incur $\geq$280 cycles of sync overhead. Reduced stencil MAPE: \textbf{146\% $\to$ 3.7\%}.

  \item \textbf{Sub-linear occupancy loss} ($\alpha{=}0.15$): 3$\to$1 blocks/SM costs $\sim$16\%, not 42\%.

  \item \textbf{Parameter-free model achieves $\sim$9\% MAPE} across 4 GPU architectures, 2 kernels, zero fitted coefficients.

  \item \textbf{Triton pre-filter} prunes \cpasync{} stage candidates in autotuning search space.
\end{enumerate}
\end{block}
\end{column}
